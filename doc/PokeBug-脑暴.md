# 🪲 PokeBug — 口袋昆虫饲养 — 项目脑暴

> 基于 M5Stack StickS3 的口袋昆虫电子宠物项目
> 脑暴日期：2026-06-16
> 最后更新：2026-06-16

---

## 1. 核心概念

一个可以随身携带的昆虫饲养箱——在 StickS3 的微型屏幕上，养一只属于你的像素昆虫。通过真实硬件交互（摇晃、吹气、倾斜）与昆虫建立关系，看它从一颗卵变成威风凛凛的成虫，还能和朋友联机对战。

**一句话定位**：拓麻歌子 + 真实昆虫变态发育 + 硬件体感交互

**核心设计理念**：养成即备战。幼虫期的每一次喂食、蛹期的每一次安静守候，都直接塑造成虫的战斗属性。养成深度不是堆数值，而是让玩家的照顾行为有策略意义。

## 2. MVP 定义

> **核心原则**：先跑通一个昆虫的完整生命周期 + 属性养成闭环 + 对战验证。

### MVP 包含

| 功能 | 说明 |
|---|---|
| 1种昆虫（独角仙） | 从卵到成虫完整生命周期 |
| 属性系统 | 5项属性，养成行为直接影响属性成长 |
| 生命周期 | 卵 → 幼虫 → 蛹 → 成虫，4阶段各有属性成长逻辑 |
| 喂食 | 短按A投喂，饥饿度系统 |
| IMU交互 | 倾斜=昆虫爬行方向，摇晃=激怒/训练 |
| 培养缸 | 甲虫的生活空间，有基础物件 |
| 菜单系统 | 长按A呼出，极简图标菜单 |
| 物品系统 | 最少物品集：树汁 + 腐木 |
| 属性对战 | 基于完整属性公式的自动战斗 + A键积极性 |
| 存档 | NVS保存，断电不丢 |
| ESP-NOW通信 | 1v1直连对战（广播发现→配对→对战） |

### MVP 不包含（后续迭代）

- ❌ 多种昆虫（蝴蝶/螳螂/瓢虫等）
- ❌ 交配与遗传系统
- ❌ 训练小游戏
- ❌ 天气/环境系统
- ❌ 图鉴/收集
- ❌ 音效（StickS3无扬声器）

- ❌ 麦克风交互

## 5. 属性体系（养成×对战核心）

> 这是整个游戏的脊梁。5项属性同时服务于养成和对战，养成阶段的照顾方式直接决定属性成长，属性最终在对战中体现价值。

### 5.1 属性一览

| 属性 | 缩写 | 范围 | 含义 | 养成来源 | 对战作用 |
|---|---|---|---|---|---|
| **体型** | SIZ | 1-10 | 甲虫的体格大小 | 幼虫期喂食 | HP加成 + 伤害倍率 |
| **力量** | STR | 1-10 | 甲虫的推力/冲撞力 | 幼虫期喂食 + 成虫摇晃训练 | 基础伤害 |
| **耐久** | END | 1-10 | 甲虫的持久力/防御力 | 成虫期喂食 + 腐木休息 | 总HP + 防御减伤 |
| **斗志** | SPI | 1-10 | 甲虫的战斗意志/抗压能力 | 蛹期静置 + 对战经验 | 暴击率 + 抗动摇 |
| **积极性** | MOT | 0-100 | 当前战斗热情（状态值，非永久属性） | 对战中按A + 战前摇晃 + 日常互动 | 伤害倍率 |

### 5.2 个体基因系统

> 每只甲虫出生时携带一组**基因**，基因决定属性的成长潜力、上限和视觉特征。基因在卵阶段随机生成，**不可通过养成改变**，但可通过繁殖传递和变异。这是为后续繁殖系统预留的核心接口。

#### MVP中基因的定位：隐藏数值，感受差异

> ⚠️ MVP不含繁殖系统，基因对玩家来说是"不可操作"的。为了避免基因成为死数据，MVP采用**"感受型"设计**：

- **隐藏基因数值**：信息页不显示VIG/ATK/MNT数值，只显示属性条
- **孵化时给直觉提示**：根据基因总值给出1句话评价，让玩家"感觉"到个体差异
  - 基因总值≥30 → "这只幼虫活力十足！"
  - 基因总值20-29 → "这只幼虫看起来很健康"
  - 基因总值10-19 → "这只幼虫有点安静..."
  - 基因总值<10 → "这只幼虫...似乎不太好养"
- **APP外观基因保留展示**：颜色差异玩家看得见，知道"我的甲虫和你的不一样"
- **v0.3繁殖版本再暴露基因数值**：届时基因有了操作意义（选种、配对），信息页才显示VIG/ATK/MNT

> 💡 这样基因在MVP中**不是4个无意义数字，而是可感知的个体差异**——同样精心养，有的甲虫就是长得快，有的就是长得慢，玩家会好奇"为什么"，为繁殖系统埋下好奇心种子。

#### 基因结构

每只甲虫有4组基因，每组用1字节编码（高4位=显性，低4位=隐性），共4字节：

```
基因组结构:
┌──────────┬──────────┬──────────┬──────────┐
│ 体力基因  │ 攻击基因  │ 精神基因  │ 外观基因  │
│ VIG gene  │ ATK gene  │ MNT gene  │ APP gene  │
└──────────┴──────────┴──────────┴──────────┘
每组: [显性4bit | 隐性4bit]
```

#### 基因详解

| 基因 | 影响属性 | 显性效果 | 隐性效果 | 说明 |
|---|---|---|---|---|
| **体力基因 VIG** | SIZ, END | SIZ/END成长倍率 | SIZ/END上限修正 | 决定甲虫能长多大 |
| **攻击基因 ATK** | STR | STR成长倍率 | STR上限修正 | 决定甲虫能多大力 |
| **精神基因 MNT** | SPI | SPI成长倍率 | SPI上限修正 | 决定甲虫能多顽强 |
| **外观基因 APP** | 视觉表现 | 主色调 | 花纹/角型 | 决定甲虫长啥样 |

#### 基因如何影响养成

基因不直接给属性加分，而是**修正养成行为的效果和上限**。同样喂6次，好基因的甲虫长得更多。

```c
// 基因对成长的修正
float siz_growth_mult = 0.8 + dominant_vig * 0.1;   // VIG显性: 0.8~1.7倍
float str_growth_mult = 0.8 + dominant_atk * 0.1;   // ATK显性: 0.8~1.7倍
float spi_growth_mult = 0.8 + dominant_mnt * 0.1;   // MNT显性: 0.8~1.7倍

// 基因对上限的修正
int siz_cap = 6 + (dominant_vig + recessive_vig) / 2; // 基础上限6, 基因可推到10
int str_cap = 6 + (dominant_atk + recessive_atk) / 2;
int spi_cap = 6 + (dominant_mnt + recessive_mnt) / 2;
// END上限固定10（成虫期持续成长，不受基因限制）

// 属性成长时
actual_growth = base_growth * growth_mult;
if (attr + actual_growth > cap) actual_growth = cap - attr;
```

#### 基因值对照表（4bit = 0~15）

| 基因值 | 成长倍率(显性) | 上限贡献(显性+隐性/2) | 稀有度 |
|---|---|---|---|
| 0-3 | ×0.8~1.1 | 上限6~7 | 常见(60%) |
| 4-7 | ×1.2~1.5 | 上限8~9 | 普通(25%) |
| 8-11 | ×1.6~1.9 | 上限10 | 稀有(12%) |
| 12-15 | ×2.0~2.3 | 上限10+溢出 | 极稀有(3%) |

> 💡 **野生甲虫**（初始卵）的基因完全随机，大部分是0-7的普通基因。想要高基因，得靠繁殖。

#### 外观基因 APP — 视觉差异化

APP基因决定甲虫的外观，让同一种甲虫看起来不同。MVP只需支持主色调变化，后续可扩展花纹和角型。

| APP显性值 | 主色调 | 示例 |
|---|---|---|
| 0-3 | 标准深棕 | 普通独角仙 |
| 4-7 | 深绿/深红 | 常见色变 |
| 8-11 | 金色/蓝色 | 稀有色 |
| 12-15 | 白化/虹彩 | 极稀有色 |

```c
// 像素渲染时根据APP基因调整调色板
uint16_t base_palette[4] = {0x0000, 0x4208, 0x8410, 0xC618}; // 深棕标准色
// 根据APP显性值偏移色相
apply_hue_shift(base_palette, app_dominant);
```

> 像素图用4色调色板索引，渲染时根据APP基因替换调色板即可实现颜色变化，**不需要为每种颜色画单独的精灵**。

#### 基因与养成策略的交互

| 场景 | 效果 |
|---|---|
| 好基因+精心养成 | 属性拉满，接近上限10 |
| 好基因+放养 | 潜力浪费，实际属性低 |
| 差基因+精心养成 | 成长倍率低，但能到普通水平(6-7) |
| 差基因+放养 | 最弱甲虫，但对战时MOT操作仍有机会翻盘 |

> 💡 **基因决定天花板，养成决定到几楼**。好基因不养成=浪费，差基因好好养=能打。但好基因+精心养成=真正的王者。这给了繁殖系统最核心的驱动力：**想要更强，就得繁殖出更好的基因，然后精心养**。

#### 基因为繁殖预留的接口

```c
// 繁殖时基因遗传规则（后续实现，MVP仅存数据）
// 1. 显性/隐性随机分配：父母各出1个基因（显性或隐性），子代随机选谁显性
// 2. 变异概率：5%几率某基因+1或-1
// 3. APP基因：混合父母色调+随机偏移
//
// 示例：
//   父: VIG=[5|2]  母: VIG=[3|8]
//   子: 可能结果 → [5|3] [5|8] [2|3] [2|8] 各25%
//   其中显性位置随机 → 50%概率5显性, 50%概率3或8显性
//   变异5%: 某个值+1 → [6|3] 等
```

### 5.3 属性总览：养成阶段对应

```
            幼虫期决定          蛹期决定          成虫精修
           ┌────────┐        ┌────────┐       ┌────────┐
  SIZ ──── │  喂食   │────────│   —    │───────│   —    │  (定型)
  STR ──── │  喂食   │────────│   —    │───────│ 摇晃训练│  (可微调)
  SPI ──── │  怕震   │────────│  静置   │───────│  对战   │  (可成长)
  END ──── │   —    │────────│   —    │───────│喂食+休息│  (主要成长)
  MOT ──── │   —    │────────│   —    │───────│ 互动操作 │  (战斗状态)
           └────────┘        └────────┘       └────────┘
```

**核心设计思想**：
- **SIZ + STR 在幼虫期定型** → 模拟真实独角仙生物学（幼虫营养决定成虫体型）
- **SPI 在蛹期定型** → 蛹期的安宁决定甲虫的战斗意志
- **END 在成虫期成长** → 持续的照顾让甲虫更结实
- **MOT 是战斗时的状态值** → 玩家操作直接影响，模拟"主人给甲虫打气"

### 5.4 各阶段属性成长详情

#### 🥚 卵期（~5分钟）— 无属性成长

- 所有属性初始值：SIZ=1, STR=1, END=1, SPI=1, MOT=50
- 摇晃会延长孵化时间（每摇+30s），不影响属性

#### 🐛 幼虫期（~30分钟）— 决定型成长期

> 这是最关键的养成期。幼虫期的照顾质量直接决定成虫的SIZ和STR上限。懒散养出来的甲虫就是小弱鸡，用心养的就是大力士。

| 行为 | 属性影响 | 说明 |
|---|---|---|
| 每次喂食（短按A放树汁） | STR +0.3, SIZ +0.15 | 喂得多=壮 |
| 被剧烈摇晃检测到 | SPI -0.2 | 幼虫受惊，斗志受损（仅检测加速度>2.0g，过滤日常颠簸） |
| 饥饿度持续<30超过2分钟 | SIZ -0.1/分钟 | 营养不良影响发育 |
| 存活到化蛹 | 基础奖励：STR +1, SIZ +0.5 | 活下来就有底子 |

**幼虫期属性成长估算**：

| 养成风格 | 喂食次数 | 最终STR | 最终SIZ | SPI | 说明 |
|---|---|---|---|---|---|
| 精心照料 | 6次 | 1+1.8+1=3.8→**4** | 1+0.9+0.5=2.4→**2** | 1（无摇晃） | 大力士型 |
| 正常养成 | 4次 | 1+1.2+1=3.2→**3** | 1+0.6+0.5=2.1→**2** | 1 | 均衡型 |
| 放养 | 2次 | 1+0.6+1=2.6→**3** | 1+0.3+0.5=1.8→**2** | 1 | 瘦弱但斗志完整 |
| 频繁误摇 | 4次 | 3 | 2 | 1-1.5=**1** | 摇了3次，斗志崩了 |

> 💡 **策略选择**：喂得勤=大体型力量型，但频繁操作可能导致误触摇晃；放养=小体型但斗志完整。**没有完美养成，只有取舍**。

> 💡 **幼虫期初始6份树汁**，刚好覆盖"精心照料"6次喂食，没有余量让玩家紧张——你要么精心养成用完，要么省着点留余地。**策略从第一份树汁就开始了**。

#### 🫘 蛹期（~10分钟）— 斗志定型期

> 蛹期是"考验耐心"的阶段。安静等待=高斗志，手贱摇晃=斗志崩塌。

| 行为 | 属性影响 | 说明 |
|---|---|---|
| 安静度过全程 | SPI +3 | 在安宁中积蓄斗志 |
| 每次被剧烈摇晃 | SPI -1 | 蛹期受扰严重打击斗志（加速度>2.0g才算） |
| 蛹期结束时 | 获得1块腐木 | 道具奖励 |

**蛹期斗志结算**：

| 蛹期表现 | 摇晃次数 | SPI变化 | 最终SPI（假设幼虫期末SPI=1） |
|---|---|---|---|
| 完美安静 | 0 | +3 | **4** |
| 轻微干扰 | 1 | +3-1=+2 | **3** |
| 忍不住摇 | 2 | +3-2=+1 | **2** |
| 疯狂摇晃 | 3+ | +3-3=+0 | **1** |

> 💡 **策略意义**：高斗志的甲虫在对战中暴击率高、不容易被对手打崩——一个安静的蛹期比什么都重要。

#### 🪲 成虫期（持续）— 精修期

> 成虫的SIZ基本定型（幼虫期定的），STR可微调，END和SPI是主要成长方向。

| 行为 | 属性影响 | 冷却/限制 | 说明 |
|---|---|---|---|
| 每次喂食 | END +0.15, 饥饿度+30 | 5分钟CD | 持续喂养积累耐久 |
| 腐木休息≥2分钟 | END +0.3 | 每次休息1次 | 休息是最好的训练 |
| 摇晃设备（成虫） | STR +0.1, MOT +20 | 3分钟CD | 刺激训练，力量微增 |
| 对战胜利 | SPI +0.5 | — | 胜利是最好的兴奋剂 |
| 对战失败 | SPI +0.2 | — | 失败也有经验 |
| 戳甲虫（短按B） | MOT +5 | 30秒CD | 日常互动 |
| 饥饿度<30 | MOT上限=50 | — | 饿了没力气兴奋 |
| 饥饿度<10 | STR/END战斗临时-2 | — | 虚弱惩罚 |

**成虫期属性成长节奏**：
- END：每天（假设每天互动1小时）约可成长 +1.8（4次喂食+2次休息）
- STR：每天约可成长 +0.2（摇晃训练2次），成长缓慢，主要靠幼虫期底子
- SPI：每天约可成长 +0.7（假设1胜1负）
- SIZ：完全定型，不可成长

### 5.4 饥饿度系统

```
饥饿度: 100 ──────→ 0
         ↓喂食+30    ↓持续5分钟不喂→死亡
```

- 每30秒自然下降1点
- 喂食+30（上限100）
- 饥饿度<20时LED闪烁提醒
- 饥饿度=0后5分钟仍不喂食→死亡
- **饥饿度对属性的影响**：
  - 饥饿度<30 → MOT上限降为50
  - 饥饿度<10 → 战斗中STR和END临时-2

### 5.5 死亡与重置

> ⚠️ 这是MVP**必须包含**的流程——养死甲虫后设备不能变砖。

**死亡流程**：
1. 饥饿度=0持续5分钟 → 甲虫死亡（`alive=0`）
2. 死亡画面：甲虫翻倒+❌标记，持续显示
3. LED慢闪（红色提示）

**重置机制**：长按A+B 3秒 → 确认重置（屏幕闪烁3次）→ 清除存档 → 重新从卵开始

**重置后保留**：世代数（generation）+1（标记这是第几只虫）
**重置后清除**：所有属性、背包、战绩、基因重新随机生成

> 💡 世代数+1的设计让"死了"不是纯粹的归零——"我这是第3只虫了"有一种延续感，也为未来的图鉴/成就系统留了口子。

## 9. 对战系统（属性驱动）

### 9.1 核心设计理念

对战不是纯随机比大小，而是**养成成果的检验**。你花心思养的甲虫，属性上就是比放养的要强。但同时，**对战中的操作（按A加油、战前摇晃）也能通过积极性系统影响战局**，让操作有即时反馈。

### 9.2 战斗属性推导

属性不是直接等于战斗数值，而是通过公式推导，让不同属性组合产生不同的战斗风格：

```
┌─────────────── 养成属性 ───────────────┐     ┌──── 战斗数值 ────┐
│                                         │     │                  │
│  SIZ (体型)  ─────┬──→ HP = 20 + SIZ×3  │     │  总HP            │
│                   └──→ 伤害倍率          │     │  伤害×(1+SIZ/20) │
│                                         │     │                  │
│  STR (力量)  ────────→ 基础伤害          │     │  STR×(1+SIZ/20)  │
│                                         │     │  ×MOT倍率        │
│  END (耐久)  ─────┬──→ HP = +END×2      │     │  总HP            │
│                   └──→ 防御减伤          │     │  减伤=END×0.5    │
│                                         │     │                  │
│  SPI (斗志)  ─────┬──→ 暴击率            │     │  SPI×5%          │
│                   └──→ 抗动摇            │     │  被打时MOT少减   │
│                                         │     │                  │
│  MOT (积极性) ──────→ 伤害倍率           │     │  0.5+MOT/100    │
│                     + 暴击伤害加成        │     │  暴击×1.8       │
└─────────────────────────────────────────┘     └──────────────────┘
```

### 9.3 战斗公式

```c
// === 战斗数值推导 ===

// 总HP
int hp = 20 + siz * 3 + end * 2;

// 基础伤害（每回合）
int base_dmg = str * (1 + siz / 20.0);

// 积极性倍率
float mot_mult = 0.5 + motivation / 100.0;  // 范围 0.5~1.5

// 暴击
bool crit = (random(100) < spi * 5);  // SPI=4 → 20%暴击率
float crit_mult = crit ? 1.8 : 1.0;

// 实际伤害
int damage = (int)(base_dmg * mot_mult * crit_mult) - (int)(end * 0.5);
damage = max(damage, 1);  // 保底1点伤害

// 被打后动摇（MOT下降）
int mot_loss = 8 - spi / 2;  // SPI=4→MOT-6, SPI=8→MOT-4
motivation = max(0, motivation - mot_loss);
```

### 9.4 属性对战的策略深度

| 属性组合 | 战斗风格 | 优势 | 劣势 |
|---|---|---|---|
| 高STR+SIZ | 重炮型 | 伤害高，一击必杀潜力 | HP一般，持久战不利 |
| 高END+SIZ | 坦克型 | HP厚，能抗 | 伤害低，打不死人 |
| 高SPI+MOT | 暴击型 | 暴击率高，不被动摇 | 基础伤害低，暴击看脸 |
| 均衡型 | 全能型 | 没有明显短板 | 没有明显长板 |
| 低SIZ高STR | 刺客型 | 伤害倍率低但基础高，MOT高时爆发 | HP薄，容易被秒 |

> 💡 不同养成策略 → 不同属性分布 → 不同战斗风格。**没有最强build，只有最适合你养法的打法**。

### 9.5 战斗属性速查表

假设典型养成结果（属性范围1-10）：

| 养成类型 | SIZ | STR | END | SPI | HP | 基础伤害 | 暴击率 | 特点 |
|---|---|---|---|---|---|---|---|---|
| 幼虫精养+蛹安静 | 2 | 4 | 3(成虫喂) | 4 | 35 | 4.4 | 20% | 均衡偏攻 |
| 幼虫精养+蛹被摇 | 2 | 4 | 3 | 1 | 35 | 4.4 | 5% | 有力无心 |
| 幼虫放养+蛹安静 | 2 | 3 | 3 | 4 | 35 | 3.3 | 20% | 有心无力 |
| 老玩家(成虫养7天) | 2 | 4 | 6 | 7 | 50 | 4.4 | 35% | 全面强化 |

### 9.6 ESP-NOW 1v1直连流程（MVP）

```
双方都选对战 → 广播发现 → 收到对方广播 → 互相配对
→ 交换属性(battle_sync_t) → 自动对战 → 交换回合结果 → 结算

发现流程:
- 广播: esp_now_send(broadcast, {type:DISCOVER, my_mac})
- 收到: esp_now_add_peer(sender_mac), 回复 {type:DISCOVER_ACK}
- 配对成功: LED常亮, 开始对战
- 超时10秒: LED灭, 返回菜单

对战通信:
- 双方同时计算伤害(各自用对方属性), 回合结果互相发送
- 这样避免了"谁算谁不算"的主从问题
```

> 4人房间管理（房主/加入/观战/掉线处理）代码量大，且不是养成核心。
> 1v1直连已足够验证"属性养成×对战"的核心乐趣。
> v0.2再扩展房间制。

### 9.7 对战流程

```
对战开始 → 交换属性 → 计算双方HP → 3回合自动战斗 → 结算

每回合(约4秒):
  1. 双方甲虫冲锋动画（~1秒）
  2. 玩家可在冲锋动画中按A → MOT+15（每回合1次）
  3. 双方同时出伤害（按公式计算）
  4. 暴击时播放特殊动画 + 屏幕微闪
  5. 更新HP + MOT衰减（MOT-8+SPI/2）
  6. 某方HP≤0 → 战斗结束
  3回合后双方都存活 → 比剩余HP百分比

战后:
  胜方: SPI+0.5, 树汁+2
  败方: SPI+0.2, 树汁+0
  MOT重置为50
```

### 9.8 积极性（MOT）系统

| 时机 | 行为 | MOT变化 |
|---|---|---|
| 战前 | 摇晃设备 | +20（入场buff） |
| 战斗中 | 按A加油 | +15（每回合1次） |
| 每回合 | 自然衰减 | -(8 - SPI/2) |
| 被暴击 | 动摇 | -5 |
| 战斗结束 | 重置 | →50 |
| 日常 | 戳甲虫（短按B） | +5（30秒CD） |
| 饥饿度<30 | — | MOT上限=50 |
| 饥饿度≥30 | — | MOT上限=100 |

### 9.9 对战画面

```
┌─────────────────────────────────────────────────────┐
│                                                       │
│  🪲 ←────── 冲撞 ──────→ 🪲                        │
│  我方                   对方                          │
│  HP ████████░░  35     HP ██████████  50             │
│  MOT ██░░░ 40          RND 1/3                       │
│  ★ CRIT!               ★                             │
│                                                       │
```

## 10. 首次引导（MVP必须）

> ⚠️ StickS3没有扬声器、屏幕极小、无文字渲染——玩家第一次开机大概率不知道怎么操作。MVP**必须**有极简引导。

### 10.1 首次引导流程（~30秒）

首次开机（存档为空）→ 进入引导模式，4步交互教学：

| 步骤 | 画面 | 操作 | 目的 |
|---|---|---|---|
| 1 | 屏幕中央一个卵 + 倾斜箭头动画 | 倾斜设备 | 教"倾斜=甲虫爬行方向" |
| 2 | 卵 + A键闪烁 + 食物盘高亮 | 短按A | 教"短按A=喂食" |
| 3 | A键长闪动画 | 长按A | 教"长按A=菜单" |
| 4 | "🤫"图标 + 静止等待3秒 | 不操作 | 教"安静等待=蛹期关键" |

每步完成后进入下一步，4步全完→进入卵期正常游戏。

> 引导不需要文字，用动画+操作反馈就够了：倾斜了→卵晃一下（做对了）；按A了→食物盘亮一下（做对了）。

存档已有数据→跳过引导，直接进入培养缸场景。
