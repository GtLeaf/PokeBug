# PokeBug 交互样式说明

> 本文档梳理 PokeBug 中所有用户交互样式，方便后续迁移到其他项目或统一改造。  
> 命名力求一听即懂，每个样式包含：适用场景、视觉特征、按键约定、核心数据结构和迁移要点。

---

## 1. 通用按键约定

PokeBug 只有两个实体按键（A / B），通过**短按 / 长按**组合覆盖所有操作：

| 操作 | 约定含义 |
|---|---|
| 短按 A | 确认 / 执行 / 选择 / 喂食（主场景） |
| 短按 B | 下一个 / 切换 / 翻页 / 戳（主场景） |
| 长按 A | 返回主界面（培养缸） |
| 长按 B | 返回上级菜单 / 进入 Deep Sleep（主场景） |

> 所有菜单/信息页基本遵循此约定，可降低用户学习成本。

---

## 2. 交互样式总览

| 命名 | 典型场景 | 核心交互 |
|---|---|---|
| 纵向轮播菜单 | 主菜单、Box 子菜单、Wood 子菜单 | B 下翻，A 确认，选中项居中放大 |
| 设置键值列表 | 设置界面 | B 下翻，A 切换当前项的值 |
| 环形模式选择器 | 对战大厅模式选择 | B 在三项间循环，A 确认 |
| 纵向条目列表 | 对战大厅房间列表 | B 切换条目，A 加入 |
| 分页卡片 | 属性信息页 | B 翻页，A 返回 |
| 主场景直接操作 | 培养缸主场景 | A 喂食，B 戳，长按调出菜单/深睡 |
| 回合制 QTE | 对战场景 | CHARGE 阶段按 A 加油 |

## 3. 纵向轮播菜单

### 3.1 适用场景
入口菜单、功能分类菜单、带子菜单的导航。

### 3.2 视觉特征
- 列表纵向排列，**选中项固定在屏幕中心**。
- 选中项比未选中项**放大 1.15 倍**（或其他比例），颜色高亮（如黄色）。
- 未选中项向上下两侧散开，间距一致。
- 切换时 `selected` 跳变，`animSelected` 用线性插值平滑追上，产生滚动动画。
- 按距离中心远近排序绘制，确保选中项最后画在最上层。

### 3.3 按键约定
- 短按 B：下一项（到末尾循环回首项）。
- 短按 A：执行当前项。
- 长按 A：返回主界面。
- 长按 B：返回上级菜单。

### 3.4 核心数据结构
```cpp
int selected = 0;           // 当前选中索引
float animSelected = 0.0f;  // 用于平滑滚动的动画位置
static constexpr int SPACING = 42;      // 项间距
static constexpr float LERP = 0.25f;    // 插值速度
```

### 3.5 实现要点
1. `render()` 中根据 `animSelected` 计算每一项的 `y` 坐标。
2. 绘制前按 `fabsf(i - animSelected)` 排序，选中项最后绘制。
3. `onButton()` 中 B 改变 `selected`；若从末尾跳回首项，直接重置 `animSelected = 0` 避免整圈滚动。
4. 子菜单切换时保存/恢复 `lastSelected`，保持返回位置。

### 3.6 迁移建议
- 适合 3~8 个入口项；项数过多时滚动距离过大，建议改用分页或分组。
- 抽象为 `CarouselMenu` 类：传入字符串数组 + 回调即可使用。

## 4. 设置键值列表

### 4.1 适用场景
参数设置页：亮度、字体、游戏速度、idle 时间等。

### 4.2 视觉特征
- 纵向文本列表，一行一项。
- 选中项左侧有**短竖条指示器**。
- 每项右侧显示**当前值**（如亮度 `128`、速度 `2.0x`）。
- 项目之间用细灰线分隔。
- 内容超出屏幕时，通过 `scrollY` 自动滚动，保持选中项可见。

### 4.3 按键约定
- 短按 B：下一项。
- 短按 A：**切换当前项的值**（循环或加减）。
- 长按 A：返回主界面。
- 长按 B：返回上级菜单。

### 4.4 核心数据结构
```cpp
int8_t cursor = 0;          // 当前光标
int scrollY = 0;            // 纵向滚动偏移
static constexpr int ITEM_COUNT = 6;
```

### 4.5 实现要点
1. `render()` 中根据 `cursor` 计算 `scrollY`，确保选中项在可视区域内。
2. `onButton()` 中 A 键调用对应的 `cycleXxx()` 或 `adjustXxx()` 修改值。
3. 退出时 `onExit()` 统一保存设置。

### 4.6 迁移建议
- 适合配置项较多的场景；项数超过一屏时自动滚动体验好。
- 抽象为 `SettingsList` 类：每项由 `label + getter + setter/cycler` 描述。

## 5. 环形模式选择器

### 5.1 适用场景
选项少且互斥的模式选择，如对战大厅的 Create / Search / Back。

### 5.2 视觉特征
- 复用设置键值列表的绘制风格（标题 + 纵向列表）。
- 逻辑上是**首尾相连的环**：Create → Search → Back → Create。

### 5.3 按键约定
- 短按 B：在三个模式间循环切换。
- 短按 A：确认进入当前模式。
- 长按 A/B：返回菜单。

### 5.4 核心数据结构
```cpp
enum class Mode { NONE, CREATE, SEARCH, BACK };
Mode selectedMode = Mode::NONE;

static Mode nextMode(Mode m) {
    switch (m) {
        case Mode::SEARCH: return Mode::BACK;
        case Mode::BACK:   return Mode::CREATE;
        default:           return Mode::SEARCH;
    }
}
```

### 5.5 实现要点
1. 用枚举表示模式，`nextMode()` 实现环形切换。
2. 绘制时将枚举映射为 `cursor` 传给通用列表绘制函数。
3. 确认后进入对应子状态机（等待加入 / 扫描房间 / 返回）。

### 5.6 迁移建议
- 适合 2~4 个互斥选项；选项再多建议用纵向轮播菜单。
- 可与纵向轮播菜单合并为同一种组件，环形只是索引计算方式不同。

## 6. 纵向条目列表

### 6.1 适用场景
展示动态数据列表，如搜索到的对战房间列表。

### 6.2 视觉特征
- 与设置键值列表同风格：标题 + 纵向列表 + 左侧指示条 + 分隔线。
- 条目内容由运行时数据生成（如 `ROOM 001`、`ROOM 042`）。

### 6.3 按键约定
- 短按 B：在条目中切换。
- 短按 A：选中并执行（如加入房间）。
- 长按 A/B：返回。

### 6.4 核心数据结构
```cpp
uint8_t selectedRoomIdx = 0;
uint8_t roomCount = 0;
```

### 5.5 实现要点
1. 列表数据由外部模块（如 `BattleLink`）动态更新。
2. 每次进入列表前重新统计 `roomCount`，防止索引越界。
3. 列表为空时自动回到扫描状态或显示提示。

### 6.6 迁移建议
- 本质是设置键值列表的只读/执行版，去掉右侧值显示即可复用。
- 适合好友列表、存档列表、日志列表等。

## 7. 分页卡片

### 7.1 适用场景
信息展示页，内容可自然分成几页，如属性页 / 战绩页。

### 7.2 视觉特征
- 每页是一个完整的信息卡片。
- 页面切换为**横向翻页**效果（通过 `page` 索引切换）。
- 右侧（或底部）有**页面指示点**，当前页高亮。
- 顶部标题栏显示当前页标题。

### 7.3 按键约定
- 短按 B：下一页（循环）。
- 短按 A：返回上级菜单。
- 长按 A/B：返回主界面 / 上级菜单。

### 7.4 核心数据结构
```cpp
int page = 0;
static constexpr int PAGE_COUNT = 2;
```

### 7.5 实现要点
1. `render()` 根据 `page` 绘制对应内容页和页面指示点。
2. `onButton()` 中 B 使 `page++`，到末页循环回 0。
3. 每页内容独立渲染函数（如 `renderStatus()` / `renderRecord()`）。

### 7.6 迁移建议
- 适合 2~5 页信息；页数再多建议用纵向列表。
- 抽象为 `PagedView` 类：传入页面数组和绘制回调。

## 8. 主场景直接操作

### 8.1 适用场景
游戏主场景、培养缸等核心玩法界面。

### 8.2 视觉特征
- 不显示菜单，按键直接作用于场景对象。
- 通过动画反馈操作结果（如喂食后食物出现、被戳后甲虫抖动）。

### 8.3 按键约定
- 短按 A：执行主交互（喂食 / 确认 / 对战加油）。
- 短按 B：执行副交互（戳 / 切换选项）。
- 长按 A：打开菜单。
- 长按 B：从子菜单返回 / 进入 Deep Sleep。

### 8.4 核心数据结构
```cpp
AdultState adultState;  // 主场景中甲虫的运动状态机
int bugX, bugY;         // 甲虫位置
```

### 8.5 实现要点
1. `update()` 处理对象状态机（如甲虫的 IDLE / WALK / EAT / TURN）。
2. `onButton()` 直接调用对象行为接口（如 `bug.placeSapInTray()`）。
3. 长按阈值统一为 500 ms，由 `ButtonDispatcher` 分发。

### 8.6 迁移建议
- 这是游戏核心场景的默认交互风格；与菜单风格互补。
- 迁移时重点抽象对象行为接口，保持按键语义一致。

## 9. 回合制 QTE

### 9.1 适用场景
战斗、节奏挑战等需要限时按键的玩法。

### 9.2 视觉特征
- 回合自动推进：CHARGE → CLASH → ROUND_END → 下一回合。
- CHARGE 阶段提示玩家按键加油。
- 按键后给出即时反馈（如 MOT +15、闪光、抖动）。

### 9.3 按键约定
- CHARGE 阶段短按 A：本回合加油一次（每回合限一次）。
- 战斗结束短按 A：返回主界面。
- 长按 A：中途退出战斗。

### 9.4 核心数据结构
```cpp
bool roundBoosted = false;  // 本回合是否已加油
uint32_t stateStartMs = 0;  // 当前状态开始时间
```

### 9.5 实现要点
1. `update()` 根据状态机推进回合，CHARGE 阶段限时 `CHARGE_MS`。
2. `onButton()` 仅在 `state == CHARGE` 且未加油时响应 A 键。
3. 每回合开始时重置 `roundBoosted = false`。

### 9.6 迁移建议
- 适合任何限时单键操作；可扩展为连打、长按蓄力等变体。
- 抽象为 `TimedAction` 类：窗口时间 + 触发回调 + 冷却标记。

## 10. 迁移到其他项目的建议

### 10.1 可复用组件抽象
| 组件 | 职责 | 输入 |
|---|---|---|
| `CarouselMenu` | 纵向轮播菜单 | 标题、字符串数组、选中回调 |
| `SettingsList` | 设置键值列表 | 设置项数组（label + 值 + 切换函数） |
| `RingSelector` | 环形模式选择 | 模式枚举、nextMode 函数 |
| `ItemList` | 纵向条目列表 | 动态字符串数组、选择回调 |
| `PagedView` | 分页卡片 | 页面数量、每页绘制函数 |
| `TimedAction` | 限时 QTE | 窗口时间、按键、触发回调 |

### 10.2 优先统一的地方
1. **绘制函数**：`LobbyScene::drawSettingsStyleList()` 与 `SettingsScene::renderMenu()` 高度相似，可合并为一个通用列表绘制器。
2. **按键语义**：统一短按 A=确认、短按 B=下一个、长按 A=回主页、长按 B=回上级。
3. **光标动画**：纵向轮播的 `animSelected` 插值可泛化为任意列表的光标平滑移动。

### 10.3 注意事项
- 屏幕分辨率 240×135，所有样式都按小屏设计；迁移到大屏需重新计算间距和字号。
- 按键消抖、长按阈值由 `ButtonDispatcher` 统一处理，迁移时需保留相同事件抽象。
