# PokeBug 美术资源绘制规范

本文档记录当前固件中美术资源的绘制规格、命名约定和代码使用方式。目标是让新增资源可以稳定进入 `src/assets/`，并在 M5StickC S3 的 240x135 屏幕上正确显示。

## 目录约定

- `asset/mainScene/`: 主界面资源原图。
- `asset/mainScene/beetle/hercules/adult/`: 赫拉克勒斯成虫动作帧原图。
- `asset/mainScene/goods/wood/`: 腐木资源。
- `asset/mainScene/goods/bowl/`: 食盆资源。
- `asset/mainScene/action/`: 交互动作资源，例如 `finger.png`。
- `asset/food/`: 食物资源。
- `src/assets/`: 固件实际使用的 C++ 资源表。
- `spareAsset/`: 处理后的资源备份、预览图和旧资源归档。

## 屏幕与颜色

- 屏幕逻辑尺寸为 `240x135`。
- 主界面右侧约 `40px` 用于状态栏，成虫活动区应避免进入右侧状态栏。
- 固件绘制使用 RGB565。原图可以是 PNG，但进入代码时会转换成 RGB565 或 RGB565 RLE。
- 带透明度的 PNG 会按 alpha 生成透明跳过段。
- 绿底图会在生成资源时抠除。建议绿底使用高饱和亮绿，避免角色或道具本身使用接近的颜色。

## 主界面背景

主界面支持两类背景资源：

- 分层资源：背景、底材、状态栏分别绘制。
- 全屏资源：文件名中带 `full`，例如 `bg_full_beginner.png`，按整张 240x135 绘制，不再区分背景、底材、状态栏。

使用方式：

- 分层资源适合复用状态栏或底材。
- 全屏资源适合完整场景图，例如房间、森林等。
- 新增背景后需要更新 `MainSceneAssets` 和 `GameEngine::MainSceneBg` 枚举，并在菜单中加入可切换项。

## 成虫动作帧

当前成虫代码使用可变帧尺寸：

- 最大帧宽：`58px`
- 最大帧高：`39px`
- 每一帧在 `RleFrame` 中记录自己的 `width/height`。
- 代码资源：`src/assets/HerculesAdultSprites.*`

动作集合：

- `WALK`: 巡逻移动。
- `EAT`: 进食。
- `TURN`: 转身过渡，每次转身随机选一张中间姿态。
- `THREATEN`: 被戳中后的威吓动作。
- `RESET`: 死亡或重置相关姿态。

绘制要求：

- 同一动作内视觉尺度要统一，不要每帧独立塞满画布。
- 角色脚部或身体底部应按同一基线对齐。
- 原图可以带透明，也可以是绿底，生成资源时会统一处理。
- 如果原始素材很大，应先抠出有效区域，再按参考动作的视觉尺度缩放；低姿态或前伸姿态可以更宽或更矮，但不能因为被塞进固定画布而整体变小。
- `THREATEN` 以甲虫身体为视觉标准，允许抬起的腿或角让画布更高。

后续生成注意事项：

- 生成脚本：`spareAsset/scripts/generate_hercules_adult_sprites.py`。
- 基准动作是 `WALK`。当前 `WALK` 视觉高度约 `28px`，身体长度约 `44-47px`，后续新增动作应优先和这个身体尺度接近。
- 不要用“每帧最大化塞满画布”的方式生成。长身体低姿态会被压矮，抬角或抬腿姿态会被压小，最终看起来像不同大小的甲虫。
- `THREATEN` 看身体，不看整体 bbox。抬起的腿、角、前足可以让画布更高；身体主体宽度应接近 `58px`，避免第三帧因为抬高而整体缩小。
- `RESET` 是趴低姿态，应比 `WALK` 更矮，但身体不应横向撑满到显得更大；当前目标约 `44x18`。
- `TURN` 是正面/背面姿态，天然比侧面窄；当前目标宽度约 `30-32px`、高度约 `30px`，不需要强行拉到侧面宽度。
- `EAT` 是低头进食姿态，允许高度低于 `WALK`；当前目标宽度约 `50px`、高度约 `20-22px`。
- 所有帧必须底部基线一致。生成预览图里的红线是脚底/身体底部参考线，帧底部应贴近红线。
- 新增动作后必须检查 `spareAsset/previews/beetle/hercules/adult/hercules_adult_variable_sheet_preview.png`，用肉眼确认身体尺度、底线、朝向和帧间跳动。
- 如果新增动作宽度超过当前最大帧宽，需同步检查 `TerrariumScene::MAX_X`，避免成虫进入右侧状态栏。
- 生成后同步检查 `src/assets/HerculesAdultSprites.h` 中的 `MAX_FRAME_W/MAX_FRAME_H`，并运行 `pio run` 验证资源体积和编译。

## 道具与食物

当前主界面道具规格：

- 腐木：`WoodAssets`，当前帧尺寸 `87x59`。
- 食盆：`BowlAssets`，当前帧尺寸 `48x24`。
- 食物：`FoodAssets`，当前帧尺寸 `32x16`。

摆放规则：

- 食盆紧贴底材，位于左侧。
- 食物绘制在食盆上方，并根据剩余量做不规则裁切，模拟被吃掉。
- 腐木紧贴底材，位于右侧但不遮挡状态栏。
- 腐木尺寸按成虫最大参考尺寸 `58x39` 的 `1.5x` 生成，当前为 `87x59`。
- 腐木生成脚本：`spareAsset/scripts/generate_wood_assets.py`。后续只改脚本顶部的 `ADULT_REFERENCE_W`、`ADULT_REFERENCE_H` 或 `WOOD_SCALE` 即可整体调整大小。
- 食盆生成脚本：`spareAsset/scripts/generate_bowl_assets.py`。后续只改脚本顶部的 `BOWL_FRAME_W`、`BOWL_FRAME_H` 即可整体调整大小。
- 食物生成脚本：`spareAsset/scripts/generate_food_assets.py`。后续只改脚本顶部的 `FOOD_FRAME_W`、`FOOD_FRAME_H` 即可整体调整大小。
- 食盆和食物源图应让每个款式保持独立连通区域；透明底或高饱和绿底都可以，生成时会自动抠除背景并按从左到右顺序编号。
- 道具尺寸应参考成虫主视觉尺度，腐木约为成虫 `1.5x`，食盆低矮但能容纳食物。

## 交互动作资源

`asset/mainScene/action/finger.png` 用于主界面戳碰反馈。

当前代码生成规格：

- 代码资源：`src/assets/ActionAssets.*`
- 手指最大帧尺寸：`39x29`
- 每一帧在 `RleFrame` 中记录自己的 `width/height/tipX/tipY`。
- 角度帧数：`3`
- 原图朝右，运行时用 `flipX` 生成从右侧戳入。

使用方式：

- 按 B 成功戳中时，随机选择左侧或右侧大方向。
- 每次随机选择一个角度帧，并带少量上下偏移。
- 手指会先向目标推进，再回收，形成一次“戳”的位移动画。
- 成虫被戳时会朝向戳来的方向播放 `THREATEN`。

## 资源压缩方式

主界面彩色资源优先使用 RGB565 RLE：

- 透明区域编码为透明 run。
- 可见像素编码为 RGB565 run。
- 适合透明背景、像素块连续、空白较多的精灵。

节省空间的建议：

- 保持透明区域干净，避免半透明噪点。
- 同一动作帧使用相同画布和相近位置，减少视觉跳动。
- 不要为了压缩强行增加帧数。帧数越多通常资源越大，除非新增帧复用度极高且有专门的差分编码。目前代码使用 RLE，不做跨帧差分。
- 对于全屏背景，RLE 不一定总是更小，应按实际图像测试。

## 新增资源流程

1. 将原图放入对应 `asset/` 目录。
2. 确认尺寸、透明或绿底、帧排列方式。
3. 生成 `src/assets/*.h` 和 `src/assets/*.cpp`。
4. 在对应场景中 include 资源头文件并调用 `PixelRenderer` 绘制。
5. 生成预览图放入 `spareAsset/previews/`。
6. 处理后的拆帧图放入 `spareAsset/extracted/`。
7. 运行 `pio run` 验证编译和资源体积。
